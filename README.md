# 🛡️ The Paranoid Minimalist

> *The best code is the code I didn't write. The second best is the code no one can misuse.*

A reusable AI coding skill that combines **YAGNI**, **production-grade correctness**, and **minimal cognitive complexity**.

Instead of optimizing for the shortest code, **The Paranoid Minimalist** optimizes for the **simplest implementation that is difficult to break, easy to review, and cheap to maintain**.

---

## Philosophy

Every line of code is a future maintenance cost.

This skill teaches an AI to think like a senior engineer who is:

* Too lazy to write unnecessary code.
* Too paranoid to trust unnecessary complexity.

The result is code that prefers deletion over addition, reuse over reinvention, and correctness over cleverness.

---

## Core Principles

* 🪓 Delete before adding.
* ♻️ Reuse before rebuilding.
* 📚 Prefer the standard library.
* 🧩 Minimize cognitive complexity.
* 🛡️ Validate every trust boundary.
* 🚫 Never sacrifice correctness for brevity.

---

## Decision Process

The skill follows a fixed decision ladder:

1. State critical assumptions.
2. Apply YAGNI.
3. Delete obsolete code.
4. Reuse existing code.
5. Prefer the standard library.
6. Use existing project dependencies.
7. Leverage native platform features.
8. Write the simplest readable implementation.

It stops as soon as a complete solution exists.

---

## Safety Checks

Before considering any solution complete, the skill verifies:

* Edge cases
* Input validation
* Trust boundaries
* Failure modes
* Performance sanity
* Determinism
* Idempotency
* Resource cleanup
* Precision-sensitive calculations

Finally, it asks one last question:

> **"What can I delete?"**

## Example

### Prompt

```text
Use The Paranoid Minimalist (full).

Implement an API endpoint that creates a user account.
```

### Expected Behavior

Instead of immediately writing code, the model will:

* Eliminate unnecessary abstractions.
* Reuse existing utilities.
* Prefer standard library features.
* Validate inputs.
* Consider failure cases.
* Avoid accidental complexity.
* Produce concise, production-ready code.
